This guide deploys Oxy on a single GCP Compute Engine VM using `oxy start`, which manages PostgreSQL and the Oxy process for you.

## 1. Create a VM

In the [GCP Console](https://console.cloud.google.com/) → **Compute Engine** → **VM instances** → **Create instance**:

| Setting | Value |
|---|---|
| **Machine type** | `e2-standard-2` (2 vCPU, 8 GB) — scale up as needed |
| **Boot disk** | Ubuntu 22.04 LTS, 20 GB |
| **Firewall** | Allow HTTP and HTTPS traffic |

Note the **External IP** once the VM is running.

<Note>
  Open port 3000 in your firewall rules if you want to access Oxy directly without a reverse proxy. Go to **VPC network → Firewall** and add a rule for TCP port 3000.
</Note>

## 2. Connect and install

SSH into your VM (use the **SSH** button in the console, or `gcloud compute ssh`), then install Docker and Oxy:

```bash
# Install Docker
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
newgrp docker

# Install Oxy
curl --proto '=https' --tlsv1.2 -sSf https://get.oxy.tech | sh
```

## 3. Set up your project

Clone your Oxy project repository or create a new one:

```bash
# Clone existing project
git clone https://github.com/your-org/your-oxy-project.git
cd your-oxy-project

# Or initialize a new one
mkdir my-oxy && cd my-oxy
oxy init
```

## 4. Start Oxy

```bash
oxy start
```

`oxy start` launches PostgreSQL in Docker and starts the Oxy server. Your instance is now running at `http://YOUR_VM_IP:3000`.

<Tip>
  To keep Oxy running after you close your SSH session, use a terminal multiplexer:
  ```bash
  screen -S oxy
  oxy start
  # Detach with Ctrl+A, D
  ```
</Tip>

## 5. (Optional) Use an external database for production

When you're ready to use a managed database instead of the Docker-managed PostgreSQL:

```bash
export OXY_DATABASE_URL="postgresql://user:password@your-cloudsql-host:5432/oxy"
oxy serve
```

Use [Cloud SQL for PostgreSQL](https://cloud.google.com/sql/docs/postgres) for a fully managed option with automated backups.

## Recommended VM sizes

| Team size | Machine type | Memory |
|---|---|---|
| 1–3 users | `e2-medium` | 4 GB |
| 3–10 users | `e2-standard-2` | 8 GB |
| 10+ users | `e2-standard-4` | 16 GB |
